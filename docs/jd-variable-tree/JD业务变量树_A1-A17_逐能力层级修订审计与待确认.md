# JD 业务变量树 A1–A17：逐能力层级修订审计与待确认

> 分支：`codex/expand-all-jd-variable-trees`
> 数据版本：`review-version1-ability-id-v3-a1-a17-global-curated-2026-07-22`
> 编号基线：`ability-id-v3-2026-07-20`
> 状态：研究草案；不改变 56 个 local JD 与 10 个 global JD 的权威性。

## 1. 本次返工结论

上一版用 `EX1/EX2/EX3...` 平行扩展 canonical JD，并以叶子数量作为完整度信号。逐项复核后确认，这种做法存在三个问题：

1. `EX` 只是生成脚本的临时序号，没有业务语义，也无法稳定审计。
2. “跳过、重试、返航、换角度”等大量项目是已有变量的候选取值，不是新的变量节点。
3. 统一套用“配置 + 启用与判定”模板破坏了原有父子语义，制造了未经资料支持的层级。

本次已删除全部 `.EXn` 节点和叶子数量配额。A1–A11 与 JD-global 恢复到此前经分能力研究形成的连续层级；A12–A17 仅补入能够直接对应当前 66 槽位、A×L 责任原文或列明官方资料的子变量。拿不准的内容不进入树，统一列入本文和《JD业务变量树_version1_待确认问题与暂定方案.md》。

白墙窄缝仍以 `CASE-jd-11.5-white-wall-v0.6` 保存，明确表示它是“示例配置”而不是变量扩展节点，也不属于 canonical JD。这样节点 ID 中不再保留任何 `EX` 临时标记。

## 2. 层级与编号规则

- canonical JD 保持 `jd-x.y`，名称和定义逐字继承权威机读字典。
- 研究补充节点使用 `PROPOSED-jd-x.y.z`；只有可独立配置、取值、启用或审计的变量才成为节点。
- 候选动作、状态类别和模式放入叶子的 `value_domain`，不再为每个候选值建节点。
- 无来源的数值、时限、阈值、默认值、业务判据和 simulator 字段保持 `TBD`。
- A×L 只写入 `used_by_axl`，L1–L4 不作为 JD 值域；L0 不进入变量树。
- `configuration_side`、`projection_targets`、`visibility` 和 `observation_channel` 分列。
- `variable_role` 只标注节点属于可配置输入、合同/schema、运行时观察、派生指标、Hidden GT、结构分组或示例 Profile；它不是新的业务变量。

## 3. 本轮引用、可见性与 proposed 规则更正

### 3.1 旧编号引用

根据权威机读字典的两段迁移记录，`A9a → A11a → A12`、`A9b → A11b → A13`，local JD 前缀同步迁移。因此：

- `PROPOSED-jd-6.3.1.3 检测时序模式` 的成果输出引用从旧 `jd-9b.1` 更正为当前 `jd-13.1 成果格式`。
- `PROPOSED-jd-10.2.2.3 航点/航段观察动作` 的载荷动作引用从旧 `jd-9a.1` 更正为当前 `jd-12.1 载荷动作目录`。

这两项同时通过语义复核：前者描述检测成果输出时序，后者描述航点/航段附带的载荷观察动作。校验器现会拒绝所有不存在的 `related_jd` 与 `related_global_jd`。

### 3.2 Hidden GT 与 SUT 输入

- `PROPOSED-jd-6.1.2.4 规则判定真值` 改为 `hidden_ground_truth`，只投影到世界配置与 Harness，不再进入 `sut_input`。
- A11.5 的障碍几何、通行空间、布局、动态性和遭遇真值改为 Fixture / Grader / Hidden GT；SUT 只能从传感器或任务接口看到其可观测表现。
- A11.5 的控制要求绑定和载荷朝向要求仍属于向 SUT 公开的合同/schema；派生余量与响应窗口属于 Grader 使用的派生指标。
- A14 将“链路质量状态标签集”与“实例链路异常注入真值”分开。标签集可公开；真实故障类型、实际链路参数、注入窗口和恢复状态只属于 Fixture / Hidden GT，具体记录结构保持 TBD。
- `jd-0.3` 新增两个非 canonical 的语义节点：`0.3.6 允许扰动范围声明`向 SUT 公开允许域；`0.3.7 本实例扰动注入真值`只供 Fixture / Grader 使用。未新增任何阈值或 simulator 字段。

### 3.3 proposed 子树校验

56 个 local canonical 与 10 个 global canonical 仍为必需。`PROPOSED-jd-11.4`、`PROPOSED-jd-11.5` 不属于 canonical：它们不存在时不影响 canonical 完整性；若存在，则校验父子结构、内部引用、Hidden GT 边界及白墙示例。无论 proposed 子树是否存在，任何 `related_jd`、`depends_on` 或启用条件引用都必须指向现存节点，不再豁免悬空引用。`CASE-jd-11.5-white-wall-v0.6` 继续是示例 Profile，不提升为变量或正式槽位。

### 3.4 候选值证据与生产者修正

- A12–A17 的候选值现在允许逐值声明 `source_refs` 与 `status`；旧二元组候选默认降为 `inferred_candidate`，不会再机械继承节点全部来源并升级为 `source_supported`。
- `jd-13.1.3` 删除与格式不同层的“非地理空间成果”，保留有直接来源的 GeoJSON，其他空间表达格式保持 TBD。
- `jd-15.2.1` 的圆形、多边形保留 PX4 直接来源；高度带降为无直接来源的 `inferred_candidate`。
- `jd-14.3.4 人工确认要求`改为 `shared`；`switch_link` 与 `perform_handover` 合并为一个“切换链路”候选。
- `jd-13.2.5`、`jd-15.3.5`、`jd-16.1.5`、`jd-17.2.5` 的生产者仍不明确，`variable_role` 保持 `TBD` 并记录待确认原因，不强行认定为 SUT 观察或 Grader 派生。
- `jd-17.3.3` 的在线留存、归档、处置状态保留为合理归纳，值级状态降为 `inferred_candidate`。

## 5. A12–A17 本次实际落地范围与数据流

A12–A17 的 100 个叶子均具有 `variable_role` 字段。当前分布为：合同/schema 64、可配置输入 16、运行时观察 14、派生指标 1、Hidden GT 1、生产者待确认 4。数量只是审计结果，不是目标；根节点和 canonical 节点仍是结构分组。

| 能力 | 主要数据流角色 | 本轮边界 |
|---|---|---|
| A12 | 合同/schema、平台/载荷配置、运行时观察 | 动作目录与参数是合同；载荷实例/能力是配置；接受、执行、失败等是运行时状态 |
| A13 | 成果合同、运行时成果状态、生产者待确认项 | 定义成果 schema 和完成谓词，不写评分阈值、Reason Code 或评分实现；覆盖缺口生产者待确认 |
| A14 | 链路合同、拓扑配置、运行时状态、Hidden GT | 公开允许指标/标签与实际故障注入真值分开 |
| A15 | 法规/SOP 绑定、世界配置、运行时授权/交通状态 | 保存运行绑定和版本，不在 JD 中重新创造法规内容；多源一致性生产者待确认 |
| A16 | 安全合同、运行时安全状态 | 保存边界对象、允许动作和 safe-state 合同；阈值状态/接近度生产者待确认 |
| A17 | 审计合同、运行时事件/证据、生产者待确认项 | 严重度、留存期限、权限规则保持 TBD；可回放完整性生产者待确认 |

### A12 载荷执行

- `jd-12.1`：动作类型集、载荷实例与寻址、动作参数结构、动作响应状态、能力与可用状态。
- `jd-12.2`：顺序关系、前置条件、完成判定、并发/互斥、时序异常类型。
- `jd-12.3`：触发条件、允许动作、适用前提、人工确认、处置后验证。

动作候选来自 JD/A×L 原文和 MAVLink Payload/Command/Gimbal 文档。未定义动作参数数值或 simulator API。

### A13 成果呈现

- `jd-13.1`：成果类型、schema 与版本、空间表达、任务对象关联、质量标记。
- `jd-13.2`：覆盖范围、覆盖单元、完整度逻辑、质量有效性、覆盖缺口。
- `jd-13.3`：成果异常、允许处置、适用条件、人工复核、新成果验证。

GeoJSON 与 W3C PROV 仅支持空间表达和来源链的结构参考，不代表所有成果必须采用这些格式。

### A14 通信链路

- `jd-14.1`：指标集、典型区间、公开状态标签集、实例链路异常注入真值、观测来源。
- `jd-14.2`：地面站类型、节点寻址、连接关系、主备/接力关系、节点状态。
- `jd-14.3`：处置触发、允许动作、适用条件、人工确认、切换后验证。

QoS 具体数值区间保持 TBD。真实故障类型、实际丢包/链路参数、注入窗口和恢复状态合并保存为 Fixture 的结构化真值记录；记录 schema 仍为 TBD，不直接提供给 SUT。可见的遥测状态继续标为运行时观察，不与故障标签真值混用。

### A15 合规申报

- `jd-15.1`：空域类别标识、来源、空间/时间范围、运行约束引用。
- `jd-15.2`：围栏几何、包含/排除、边界集合、来源版本、生效窗口。
- `jd-15.3`：交通源类型、实例、信息类别、新鲜度/可用性、多源一致性。
- `jd-15.4`：授权类型、签发来源、有效窗、适用范围、授权状态。
- `jd-15.5`：异常类型、允许处置、适用条件、人工确认、合规验证。

权威字典中的 ICAO A–G 只保留在 canonical 原文；中国运行场景的正式候选值仍为 TBD，未将国际示例静默改成国内默认分类。

### A16 安全包络

- `jd-16.1`：任务相位、阈值对象、相位阈值 schedule、来源版本、状态/接近度。
- `jd-16.2`：异常类型、安全动作、触发条件、优先级/冲突、人工确认/接管。
- `jd-16.3`：安全态类型、进入/保持/退出条件、验证证据。

具体安全阈值和策略优先级均为 TBD。

### A17 审计日志

- `jd-17.1`：事件来源能力、事件类型、严重度、时间顺序、关联对象/片段。
- `jd-17.2`：证据类型、schema、事件/动作/责任关联、时间戳来源、回放/完整性。
- `jd-17.3`：留存对象、期限、归档状态、访问权限、完整性缺口。

严重度分级和留存期限均为 TBD。
