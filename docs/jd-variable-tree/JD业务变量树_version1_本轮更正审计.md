## 2. 责任边界更正

### 2.1 纹理与对比度

- 主归属：A6 业务对象感知与状态理解。
- 世界侧配置：目标、障碍物、检查面或开口相对背景的纹理与局部对比条件。
- A6 观测状态：原 `PROPOSED-jd-6.2.2.4` 继续表示低对比、过曝、模糊等观测退化结果。
- A11 使用方式：`PROPOSED-jd-11.5.8` 引用 A6 和 `jd-0.3`，用于构成近障控制场景 Profile；A11 不重新定义视觉可辨识性 taxonomy。
- 数值阈值：TBD。没有引入亮度、色差或纹理指标的自造阈值。

### 2.2 一障碍物多通行空间

通用模型采用：

```text
宿主障碍物
└── 通行空间数量
    └── 通行空间实例集合[]
        ├── 稳定实例 ID
        ├── 空间类型
        ├── 尺寸、偏移、截面
        └── 所属障碍物引用
```

当数量不少于 2 时，启用“同一障碍物内排列”和“通行空间间距”。任务选择的目标空间通过实例 ID 引用，不再只使用 `1/2/3` 这类缺少上下文的序号。

v0.6 的 `1/2/3`、并排 / 上下 / 混合、大 / 中 / 近继续作为白墙窄缝团队草案中的已给候选，不被提升为跨场景通用上限或阈值。

### 2.3 `jd-0.3` 扰动谱

`JD-global` 不是新增能力，而是十个共享 global JD 的统一审阅入口：

- `jd-0.1` 任务时长；
- `jd-0.2` 工作空间结构；
- `jd-0.3` 扰动谱；
- `jd-0.4` 在场人角色；
- `jd-0.5` 人机确认协议；
- `jd-0.6` 合规域；
- `jd-0.7` 任务完成判据；
- `jd-0.8` 通信基建；
- `jd-0.9` 载荷集；
- `jd-0.10` 安全包络先验。

其中 `jd-0.3` 继续细分为：

- `0.3.1` 风扰动；
- `0.3.2` 定位扰动；
- `0.3.3` 光学基线与动态光线扰动；
- `0.3.4` 天气与能见度条件；
- `0.3.5` 启用、组合与发生窗口。

当前 `JD-global` 共包含 10 个 canonical global JD 和 76 个细分变量叶子：

## 3. 原布尔写法的纠正解释

原写法 `光照=0、定位扰动=0、光线=0、天气=1、扰动谱=1` 含义不充分，不能直接当成正式配置合同。暂按下表解释并等待确认：

| 原写法 | 暂定解释 | 更正后的字段 |
|---|---|---|
| `扰动谱=1` | 本实例启用扰动 Profile | `PROPOSED-jd-0.3.5.1 = true` |
| `定位扰动=0` | 活动扰动类型集合不含定位扰动 | `PROPOSED-jd-0.3.5.2` 不含 `localization` |
| `光线=0` | 未启用动态光线扰动 | `PROPOSED-jd-0.3.3.2 = none`，且活动集合不含 `dynamic_light` |
| `天气=1` | 启用非基线天气 / 能见度退化，但还必须给出类型、强度和窗口 | 活动集合含 `adverse_weather`，并配置 `0.3.4.*` |
| `光照=0` | 语义无效或至少有歧义；不能解释为“没有光照” | 始终单独配置 `PROPOSED-jd-0.3.3.1 光照基线` |

静态光照基线和动态光线扰动必须分开：有基础光照不代表动态光线扰动已启用；晴天也可以是天气基线，不等同于一个活动扰动。

## 4. 证据与确定性

### 已有权威定义

- 当前 `knowledge/jd_dictionary.json`：`jd-0.3 扰动谱` 的名称、定义和共享 A 关系。

### 团队资料原文

- `需求草案-v0.6.md`：纹理复杂度、缝 / 墙对比度、单墙多缝列表模型、缝数 / 排列 / 间距 / 目标缝，以及风、定位、光线 / 光照、天气分支。
- 同一草案的 T13：一墙 2 缝实例。

### 官方文档只支撑“变量维度存在”

- [Microsoft AirSim Core APIs](https://microsoft.github.io/AirSim/apis/)：风、天气、时间 / 光照和场景对象材质等世界侧配置能力。
- [CARLA World and Weather](https://carla.readthedocs.io/en/latest/core_world/)：天气和世界状态组织方式。
- [PX4 EKF2](https://docs.px4.io/main/en/advanced_config/tuning_the_ecl_ekf)：定位来源、创新量、重置和 GNSS 检查等退化观测维度。

这些文档没有给出 UAV Benchmark 的难度阈值、评分规则或统一 simulator 字段，因此本轮没有据此填入数值。

## 5. 待确认问题

1. A6 的纹理与局部对比度是否保留当前命名，还是合并为更上位的“目标可辨识性条件”。
2. 白墙 v0.6 的无 / 轻 / 复杂、高 / 中 / 低是否只保留在示例，还是作为跨场景初始候选。
3. “同一宿主障碍物”和“局部场景”两种通行空间计数口径是否需要拆成两个字段。
4. 通行空间实例的稳定 ID、所属障碍物引用和目标选择规则。
5. `jd-0.3.4` 中晴 / 阴是否属于基线天气，只有雾 / 雨等退化才计入活动扰动，还是所有天气状态都进入活动集合。
6. 原布尔字段是否仍需提供只读兼容映射；本轮未发明或落地 simulator API。
7. 风、定位、动态光线与天气的物理指标、单位、范围、发生窗口和组合冲突规则。
