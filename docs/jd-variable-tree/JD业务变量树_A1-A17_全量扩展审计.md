# JD 业务变量树 A1–A17 全量扩展审计

> 分支：`codex/expand-all-jd-variable-trees`  
> 数据版本：`review-version1-ability-id-v3-a1-a17-global-expanded-2026-07-22`  
> 编号基线：`ability-id-v3-2026-07-20`  
> 状态：研究草案，供团队审阅；不直接改变 66 个 canonical JD 的权威性。

## 1. 本轮目标与边界

本轮把变量树范围从 A1–A11 扩展到 A1–A17，并把原先只有约 14–33 个叶子的能力补到与 A11 相近的审阅粒度。叶子数量用于发现“拆得明显不够”的分支，不是正式配额，也不代表该能力的重要性。

本轮遵守以下边界：

- 56 个 local JD 和 10 个 global JD 的编号、名称和定义逐字继承当前 66 槽位机读字典。
- A×L 只进入 `used_by_axl` 引用，不作为 JD 枚举值。
- A11 已有 82 个变量叶子和白墙窄缝示例保持不变，不把白墙案例复制到其他能力。
- 新增节点全部使用 `PROPOSED-` ID，状态为 `proposed` 或 `TBD`。
- 没有依据的数值、阈值、时限、业务判据和 simulator 映射不补值。
- “主题分组”是变量分组；真正配置和判定落在叶子。枚举候选值只允许写入 `value_domain`，不得继续做成树节点。
- `configuration_side` 与 `visibility` 分列；`visibility` 保持数组。

## 3. 统一展开模型

每个 canonical JD 下采用以下结构：

```text
能力根
└── canonical JD（权威名称与定义）
    └── PROPOSED 主题分组
        ├── 配置叶子
        └── 启用与判定叶子
```

两个叶子的职责不同：

- “配置”叶子记录 Seed 可实例化的值、集合或范围。
- “启用与判定”叶子记录 activation condition、依赖/互斥、信息来源和恢复/验证条件。

当前大部分新增 `value_domain` 为 `TBD`。这不是缺省值，而是明确表示资料尚不足，后续必须通过公开标准、论文、数据集、平台官方文档或团队受审资料补齐。

## 5. 主要公开依据

本轮新增 A12–A17 的结构参考以下官方来源；来源只支持相应概念，不自动赋予团队 canonical 权威性：

- MAVLink Command Protocol：命令参数、寻址、ACK、进行中、取消和重发。
- MAVLink Payload / Gimbal v2：载荷动作、云台能力、状态与控制权。
- MAVLink Microservices / Routing：组件寻址、通信协议族与网络拓扑。
- MAVLink Traffic Management：UTM 和 ADS-B 交通信息。
- PX4 Safety / Geofence：failsafe 动作、触发类别、围栏几何与 breach action。
- 《无人驾驶航空器飞行管理暂行条例》（国令第761号）：国内无人驾驶航空器运行管理依据。
- W3C PROV-DM：Entity、Activity、Agent 和来源链。
- OGC GeoJSON：地理空间成果的几何和属性表达。

所有 URL 已写入机读目录 `sources`，叶子通过 `source` 引用。

## 6. 待确认

1. 65–82 个叶子的详度是否适合作为第一轮统一审阅基线；是否需要对某些能力继续拆分或合并。
2. 每个 `PROPOSED` 主题是否确属 JD 业务变量，还是应迁移到 GAL、平台档案、Grader 或纯审计元数据。
3. A1/A5 哪些交互配置属于 `user_config`，哪些只能由 harness 注入或从 Trace 观察。
4. A2 平台档案与 `jd-0.9`、A9、A11 执行包络的去重边界。
5. A3/A4 内部机制哪些可以作为任务变量配置，哪些只能作为黑盒 SUT Trace 观察项。
6. A6 道路病害、车辆违法、园区对象 taxonomy 的最终裁剪与跨场景复用范围。
7. A7/A8 的坐标系、精度、完整性和恢复判据采用哪些标准或数据集口径。
8. A9 资源预警与返航储备是否由平台档案、任务需求还是安全包络提供。
9. A10/A11/A16 的责任边界：轨迹选择、闭环控制、动态执行包络和反射级安全处置如何投影。
10. A12 哪些载荷动作进入统一目录；投送、采样等高风险动作是否本轮启用。
11. A13 成果格式采用哪些标准；覆盖完整度如何区分任务需求和 Grader 判定。
12. A14 通信拓扑和链路质量中哪些是世界配置，哪些只作为 Hidden GT 或 Trace。
13. A15 国内法规、组织 SOP 与国际空域模型如何分层；任何法规阈值均需逐条核验后再填写。
14. A16 safe state 集合和分相位阈值 schedule 的正式来源与批准责任。
15. A17 证据保留时长、访问控制、脱敏和法定最低留存要求。
16. 10 个 global JD 新增子维度是否足够跨高速、园区、桥梁、油田等场景复用。
17. simulator 字段映射、默认值和具体 API 全部保持后续工作，不在本轮决定。
