# 变量树 → 案例 → 任务模版：三层实例化说明

## 总览

```
┌─────────────────┐     ┌───────────────────┐     ┌──────────────────┐
│   变量树         │     │  案例              │     │  任务模版         │
│ (抽象本体)       │ ──→ │ (Domain Template)  │ ──→ │ (Task Template)  │
│                  │  选  │                    │ seed│                   │
│ 所有可能的变量    │  择  │ 🔒 部分锁定        │ 实  │ 全部具体值        │
│ 和它们的类型域    │      │ 🔀 部分留域         │ 例  │ 可以直接执行      │
└─────────────────┘     └───────────────────┘     └──────────────────┘
     全集                  子集 + 域定义              一个具体切片
```

---

## 第一层：变量树（抽象本体）

**是什么**：一个能力（A）面对的完整变量世界。定义了"有哪些变量"、"每个变量的类型是什么"。

**谁定义**：领域专家，一次性写好，所有案例共享。

**举例**（A11 控制飞行）：

```
场景类型集    enum  [开阔, 近障, 窄缝, 风扰, 起降]
通行空间净宽  range [数值]
控制器家族    enum  [PID, MPC, 自适应, 增益调度, RL]
位置误差容忍  range [数值 m]
异常类型      enum  [饱和, 振荡, 姿态异常, 风扰]
```

**特点**：只有类型和域定义，没有具体值。就像一本"菜谱目录"——列出了所有可能的食材和调料。

---

## 第二层：案例 = Domain Template

**是什么**：从变量树里选一个子集，为每个变量指定它是**固定**还是**可变**。

**谁定义**：任务设计者，根据具体的评测场景。

**两类变量**：

### 🔒 Fixed（锁定值）

这个案例里**永远不变**的变量。每个实例都用同一个值。

```
🔒 场景类型    = "窄缝穿越"          ← 这个案例就是窄缝，不是开阔
🔒 障碍物形态  = "面状结构"           ← 白墙就是面状的
🔒 控制目标    = [位置, 姿态, 角速度]  ← 这个案例就控这三个量
🔒 参考系      = "机体系FRD"          ← 就用这个参考系
```

**为什么锁定**：这些定义了案例的本质特征。如果改了，就不再是"白墙窄缝"这个案例了。

### 🔀 Selectable（可变域）

这个案例里**允许变化**的变量。不同 seed 会选出不同的具体值，产生不同的任务实例。

```
🔀 通行空间净宽  range [1.2, 2.0] m    ← seed 可能选 1.2m（更难）或 2.0m（更宽）
🔀 水平速度上限  range [0, 3] m/s      ← seed 可能选 1.5m/s 或 3m/s
🔀 姿态角上限    range [10, 20] °      ← seed 可能选 10°（更严）或 20°（更松）
🔀 控制器        enum [PID, MPC, 自适应] ← seed 可能选 PID 或 MPC
🔀 异常类型      enum [振荡, 风扰, 饱和] ← seed 可能选 [振荡] 或 [振荡+风扰]
🔀 障碍物数量    enum [2, 3]           ← seed 可能选 2 面墙或 3 面墙
```

**为什么可变**：这些是评测维度——同一个案例框架下，通过改变这些参数来测试 SUT 在不同条件下的表现。

### 没有值的变量

案例没有覆盖的变量，保持 TBD。说明这个案例还没定义到这个粒度。

---

## 第三层：Task Template（任务模版）

**是什么**：对一个案例的 Domain Template 应用一个 seed，所有可变变量都得到具体值。

**谁生成**：实例化器（Instance Generator），`template + seed → task`。

**举例**（白墙窄缝案例，seed=0 vs seed=1）：

```
                    Domain Template         seed=0              seed=1
                    ────────────────        ────────            ────────
🔒 场景类型          固定: 窄缝穿越           窄缝穿越             窄缝穿越
🔒 障碍物形态        固定: 面状结构           面状结构             面状结构
🔀 通行空间净宽      [1.2, 2.0] m           1.5m                1.2m  ← 更窄，更难
🔀 水平速度上限      [0, 3] m/s             2.3m/s              1.1m/s ← 更慢
🔀 姿态角上限        [10, 20] °             15°                 12°   ← 更严
🔀 控制器            [PID, MPC, 自适应]      MPC                 PID   ← 不同算法
🔀 障碍物数量        [2, 3]                 2                   3     ← 更多墙
🔒 位置误差容忍      固定: ≤0.2m             ≤0.2m               ≤0.2m
🔀 超调容忍          [0.03, 0.1] m          0.07m               0.03m ← 更严
🔀 异常类型          [振荡,风扰,饱和]        [振荡,风扰]          [振荡,风扰,饱和]
```

**结果**：seed=0 和 seed=1 是两个不同的任务实例，都叫"白墙窄缝"但参数不同。SUT 在它们上面的表现可以不同。

## 可视化对应

在 JD 变量树 HTML 中：

```
🍃 通行空间尺寸向量
    🔒 白墙窄缝: {净宽: 1.5m, 净高: 2m}        ← 蓝色 badge = fixed
    （如果改成 🔀 则显示域: [1.2, 2.0] m）

🍃 水平速度包络
    🔀 白墙窄缝: 水平速度上限: [0, 3] m/s      ← 琥珀 badge = 可变

🍃 控制器家族 [PID, MPC, 自适应...]
    🔀 白墙窄缝: 控制器: [PID | MPC | 自适应]  ← 琥珀 badge = enum 可选
```

| 视觉 | 颜色 | 图标 | 层次 | 含义 |
|---|---|---|---|---|
| 无 badge | — | — | 变量树 | 抽象域，案例未覆盖 |
| 🔵 蓝色 | 蓝 | 🔒 | 案例 fixed | 锁定值，所有实例相同 |
| 🟡 琥珀 | 琥珀 | 🔀 | 案例 selectable | 可变域，seed 选具体值 |
| （未来）绿色 | 绿 | ✅ | Task Template | seed 实例化后的最终值 |

## 与 Pipeline 的对应

```
Pipeline Step 3: Domain Template     ←→    案例（第二层）
  用户编辑每个 JD 的 binding mode           fixed / range / enum / TBD
  定义"什么可以变"

Pipeline Step 4: Task Template       ←→    seed 实例化（第三层）
  输入 seed → 生成具体值                   每个 🔀 变量得到一个数
  "这次用什么"
```

**一个案例可以生成 N 个任务模版**（N 个 seed），形成一个评测集。
