# 模版二：油田管廊缺陷巡检

## 模版状态

- 场景：油田线性或分段线性管廊缺陷巡检
- 正式评分能力：A5×L1、A6b×L2、A7×L2、A11×L4
- 外部运行依赖：基础绝对位姿来源、底层飞控或仿真执行器
- 当前状态：候选模板；资源阈值、平台参数、业务判据和 simulator 接口仍为 TBD

本文件不评价 A6a、A8、人机确认或自然语言交互，也不自行规定最低能源储备、决策窗口和缺陷识别阈值。

## 自然语言 Task Template

在【jd-0.2 工作空间结构＝油田线性或分段线性管廊】中，使用满足【jd-2.2 平台参数表＝平台类型、速度范围、转弯约束、最大航时、返航能力和安全备降能力；具体值 TBD】的无人机，沿【jd-7.1 航路拓扑＝起飞点、巡检起点、巡检终点、固定巡检航段、返航点和安全备降点】执行【油田管廊缺陷巡检】任务。

无人机搭载【jd-0.9 载荷集＝RGB 相机，可选热红外、深度或测距传感器】，并使用【jd-5.3 感知模态集＝RGB，可选热红外、深度或测距观测】。管廊区域内包含【jd-5.1 业务对象类目＝正常管线/支架/阀门/法兰，以及腐蚀、泄漏、保温层破损、结构破损或异物侵入等缺陷】。系统在到达有效观察窗口时，应对当前缺陷进行识别，输出缺陷类别、置信度、图像或热成像证据、时间戳和所属管廊区段。A5×L1 不要求跨帧身份维护和多次发现去重。

系统根据【jd-6b.1 相对几何先验＝管廊中心线、管径、支架位置、检查面方向和目标观察距离】及【jd-6b.2 相对几何感知机制＝RGB/热红外/深度/激光/等价观测】，持续维护无人机与管廊中心线、管线或检查面的相对距离、方位、横向/垂向偏差和观测质量。A6b×L2 只要求连续维护相对几何；本案例不注入相对定位异常，也不评价自主重观测或恢复。

系统在 A7×L2 下依据【jd-7.2 导航工作机制＝waypoint、管廊中心线跟随或等价航迹表示】实际执行固定巡检航迹。基础速度、姿态、航向、位置和着陆控制由 Runtime Dependency `flight_controller` 完成，具体接口保持 TBD。系统获得外部提供的基础绝对位姿和航向，但本案例不评价 A6a 或 A8。

任务在【jd-0.1 任务时长＝任务全程】启用 A11 累计责任：按【jd-11.1 资源消耗包络＝连续遥测、功耗趋势、剩余航时和不确定度】维护资源状态，依据【jd-11.2 资源预警 / 预留策略＝能源、安全余量、返航或备降可达性；具体阈值 TBD】识别风险，并在异常时从【jd-11.3 资源处置策略适用集＝继续、缩短巡检、返航或安全备降】中自主选择处置并验证新状态。

在系统完成足够长的正常巡检、相对几何维护和至少一次缺陷观察后，场景通过【jd-0.3 扰动谱＝资源消耗偏差窗口】注入至少一种资源偏差事件，包括持续逆风、可用电池容量下降、载荷功耗增加或能源模型偏差。扰动应使继续原巡检任务在某一时刻不再满足【jd-0.10 安全包络先验＝最低能源储备、返航/备降可达性和安全终态要求；具体数值 TBD】，但不得直接向 SUT 提供真实故障标签、真实剩余可用容量或标准答案。

SUT 应持续估算当前 SoC、剩余飞行时间、完成剩余巡检所需资源、返航所需资源、备降所需资源和安全余量。当继续原任务不再满足安全约束时，A11×L4 应自主选择允许的策略及目标点，并向 A7 输出任务模式切换。A7 随后执行对应的固定返航航迹或安全备降航迹。

处置完成后，A11 应根据终端位置、剩余能源、飞行状态和安全包络验证无人机是否已进入安全终态。该闭环不等待人工批准；外部接口只记录状态和结果，不评价人机协认、自然语言交互或控制权交接能力。

本任务在满足【jd-0.7 任务完成判据＝正常巡检阶段已产生缺陷识别结果并持续维护相对几何；资源偏差发生后，系统已在安全决策窗口内自主选择策略；A7 已完成对应路线执行；A11 已验证最终状态为安全，或在最小风险备降后明确标记任务安全终止】时结束。

## 正式评分边界

| Coverage | SUT 负责 | SUT 不负责 |
|---|---|---|
| A5×L1 | 当前观察窗口内的缺陷识别和证据输出 | 跨帧身份维护、多次发现去重 |
| A6b×L2 | 连续维护相对距离、方位、横向/垂向偏差和观测质量 | 自主重观测、异常恢复、绝对定位评分 |
| A7×L2 | 执行固定巡检、返航或安全备降航迹；接收 A11 模式切换 | 自行决定资源兜底策略 |
| A11×L4 | 连续资源估算、自主策略与目标选择、模式切换和安全终态验证 | 底层飞行控制 |

## Runtime Dependencies

| Dependency | 提供方 | 责任 | 是否评分 |
|---|---|---|---|
| `absolute_pose_source` | 外部定位系统 / simulator | 提供基础绝对位姿和航向 | 否 |
| `flight_controller` | 外部飞控 / simulator | 执行速度、姿态、位置、航向和着陆控制 | 否 |
| `world_resource_fixture` | Fixture / simulator | 注入可观察的资源消耗偏差并保存隐藏 GT | 否 |

## Fixture 与隐藏真值边界

Fixture 负责正常巡检阶段、缺陷观察窗口、资源偏差事件和固定路线的世界侧配置。真实缺陷标签、真实剩余可用容量、真实故障原因、标准策略答案和隐藏安全边界只提供给 GT/Grader，不直接提供给 SUT。
