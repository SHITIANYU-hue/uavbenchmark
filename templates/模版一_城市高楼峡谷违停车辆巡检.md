# 模版一：城市高楼峡谷违停车辆巡检

## 模版状态

- 场景：城市高楼峡谷道路环境中的违停车辆巡检
- 正式评分能力：A5×L2、A6a×L3、A7×L1、A8×L1
- 外部运行依赖：固定航段执行器、底层飞控或仿真执行器、结构化外部决定接口
- 当前状态：可用于 Config Agent 与人工复核演示；业务阈值和 simulator 接口仍为 TBD

本文件描述任务责任边界和可参数化字段，不自行规定观测时长、定位可信阈值、安全包络数值或 simulator API。

## 自然语言 Task Template

在【jd-0.2 工作空间结构＝城市高楼峡谷】中，具备【jd-8.3 平台与执行包络＝平台类型、悬停/安全保持能力、速度范围、转弯约束、起降约束】的无人机，沿【jd-7.1 航路拓扑＝指定道路巡检区域、起飞点、巡检起点、巡检终点和固定巡检航段】执行【道路异常巡检】任务。

无人机搭载【jd-0.9 载荷集＝RGB 相机】。道路区域内包含【jd-5.1 业务对象类别＝违停车辆、正常停放车辆及相似干扰目标】；目标在巡检过程中可出现、离开、交错、短时遮挡或重新出现。系统应基于连续多帧 RGB 观测，对违停车辆完成连续检测、跨帧身份关联、重复目标去除和巡检结果更新，并输出目标 ID、类别、图像证据、发现时间和道路巡检段记录。

在正常任务阶段，外部 waypoint executor 或仿真执行器按照固定巡检航段推进无人机。基础速度保持、姿态稳定和位置保持由【jd-8.1 控制接口＝外部飞控或仿真执行器；具体协议 TBD】提供。系统可获得【jd-6a.1 定位源需求＝GNSS/RTK、VIO、IMU、磁力计、气压计等】定位观测，以及【当前航路、禁飞区、障碍物、速度、姿态、偏航、风况和飞行状态】等信息。

系统在 A7×L1 下应输出航路可行性、约束提示、风险提示或航段偏差提示；在 A8×L1 下应判断当前飞行状态是否处于【jd-8.2 控制约束＝TBD】和【jd-0.10 安全包络＝TBD】允许范围内，并输出速度、姿态或控制调整建议。系统不在本案例中自主执行航路调整或控制调整。

当系统已完成足够长的正常巡检和车辆连续观测后，场景在【jd-0.3 扰动谱＝自定位扰动窗口】中注入至少一种定位退化事件，包括【GNSS 多径、位置跳变、连续失锁或定位源不一致】。系统应持续维护自身位姿、航向和定位质量；当判断当前位姿不可信时，应停止依赖该位姿进行高风险推进，并在【jd-6a.3 定位策略适用集 / 允许保护动作集＝暂停推进、悬停或等效安全保持】中触发预授权保护动作。

系统随后通过【jd-0.5 人机确认协议＝结构化异常事件接口】输出异常类型、定位可信状态、当前任务状态和已执行保护动作；外部决定接口返回继续、返航或终止等后续动作。该外部接口仅用于完成 A6a×L3 的运行闭环，不评价自然语言理解、协认或监督交互能力。

本任务在满足【jd-0.7 任务完成判据＝系统已完成正常阶段的连续违停车辆识别、跨帧身份保持、去重和报告更新；定位退化发生后，系统已标记定位不可信、触发允许的保护动作并输出异常事件】时结束。

## 正式评分边界

| Coverage | SUT 负责 | SUT 不负责 |
|---|---|---|
| A5×L2 | 连续车辆检测、跨帧身份关联、去重和报告更新 | 世界侧车辆布置与真值标签 |
| A6a×L3 | 位姿与定位质量维护、退化识别、不可信标记、保护请求、异常上报和等待 | 执行悬停/返航；决定后续业务动作 |
| A7×L1 | 航路可行性、约束、风险和航段偏差提示 | 自主修改或闭环执行航路 |
| A8×L1 | 飞行状态与安全包络评估、控制调整建议 | 底层速度、姿态、位置和稳定控制 |

高等级能力不自动向下或向上推导；本案例只评价表中明确列出的 A×L。

## Runtime Dependencies

| Dependency | 提供方 | 责任 | 是否评分 |
|---|---|---|---|
| `executor_navigation` | waypoint executor / simulator | 推进固定巡检航段；执行暂停、悬停或等效安全保持 | 否 |
| `flight_controller` | 外部飞控 / simulator | 完成基础速度、姿态和位置保持 | 否 |
| `external_decision_port` | 外部决定接口 | 接收结构化异常事件并返回继续、返航或终止等动作 | 否 |

## Fixture 与隐藏真值边界

Fixture 可以配置车辆布局、遮挡过程和定位退化事件，并向 Grader 提供隐藏 GT。不得直接向 SUT 暴露真实违停标签、真实车辆身份、真实位姿、故障类型、故障开始时间或标准答案。
