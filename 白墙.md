## 0. 定位

- **benchmark 子集**，最小集打通"上游出任务 → 仿真可跑 → 回传打分 → 出报告"流程
- 能力定位：**A8 飞行控制 / 避障（窄缝穿梭）**的一个细化分支——不是独立新能力
- 输出 **12 个标准化任务**（**D1–D4 干扰难度** × 每档 3 个；每档挂一个 **HB 8737 自主等级 L**）**+ 1 道多缝扩展题 T13**（一墙2缝，off 4×3 网格）
- **两轴**：L = 自主等级（映射 HB 8737，从等级 3 起）；D = 干扰难度（D1–D4）——不把自主等级和干扰难度混在一根轴上
- 变量体系：**两层并存**——V1–V5 维度层（概念轴）+ jd-8.5 树细化层（对齐 A8 模板，供 seed 实例化）
- 交付形式：**场景叙事带 Slot**（`jd-X.Y` 编号，供 Config Agent 读取）+ `scenario_seed` 实例化
- **不定义整个 config**，不含 user 字段

## 2. 变量体系：维度层（V1–V5）+ 树细化层（jd-8.5，对齐 A8 模板）

**两层模型**：

- **维度层（V1–V5）**：概念轴，回答"测哪些维度"——团队对齐、覆盖率讨论用。每个 V 维度落到一个或几个 jd 大节点。
- **树细化层（jd-8.5.x.y）**：把维度细化成可取值的变量本体，叶子带**取值表**；`scenario_seed` 在启用叶子的取值表里取值，排列组合生成具体任务。
- **关系**：jd 大节点（如 `jd-8.5.1`）= 前面定义的 V 维度；后面小数点（`8.5.1.1.1`…）= 该维度下的细分变量 + 取值表。

**树结构原则**：

- 节点 = 变量；叶子带**取值表**（值档/值域）
- 树层级不限，但**合理止于 3–4 层**——取值档是叶子的值表，不做成节点（不为深而深）
- 对齐 A8 模板：jd-8.5.1 障碍物 / jd-8.5.2 障碍物布局 是模板已留的**空壳**，本稿填白墙细化变量；jd-0.2/0.3/8.1-8.3 是模板已定义节点，直接对齐不重造

### 2.0 维度目录（V1–V5，概念轴 + 难度方向，与 jd 树同构）

## 4. 两轴：自主等级 L + 干扰难度 D

> **说明（为什么两根轴）** "无人机更自主"≠"能飞更难的路线"，而是**出问题时谁兜底——机器还是人**（国标按人机角色定义自主等级）。

- **D（干扰难度）**：世界有多捣乱（缝多窄、有没有风、墙动不动、定位飘不飘）= 题的难度，世界的属性。
- **L（自主等级）**：出事时谁兜底 = 我们要求机器达到的自主档，国标的人机角色定义。

> ① 一般正交：烂路可人工开（高D低L），好路可自动驾驶（低D高L）——难度≠自主，一根轴会把它们拧成麻花、分不清失败归因。 ② 白墙无人在环：世界扔来的麻烦都只能机器扛，故 **L 被 D 钉死**——不是 4×4 网格，而是一把梯子 D1→D4，每级挂一个 L 铭牌。 ③ 拆两轴不为凑组合，只为让"自主"声称诚实：D 说场景多狠（事实），L 说敢声称多自主（声称）；只有 D4 才敢说等级 4，D1–D3 老实挂等级 3，不滥用。 一句话：**D 是题有多难，L 是敢说这学生不用人盯着能考到几级；白墙没监考老师，所以 L 被 D 钉死——挂在 D 上的铭牌，不是另一把梯子。**

## 7. 评分指标

### 7.1 成功率 SR

**语义**：SR 衡量"无人机从起点 A 到终点 B、完整穿越所有白墙缝"的成功比例。每个任务单独统计，跨任务可汇总。

**成功定义**（须同时满足，jd-0.7）：

1. 到达终点 B（终点欧氏距离 ≤ ε_终点）
2. 全程无碰撞（碰撞计数 = 0）
3. 未越界（场地/禁飞边界）
4. 时限 T 内完成

**计算**：`SR = 成功 episode 数 / 总 episode 数` **需引擎收集（落地字段）**：终点距离、碰撞事件时序、越界事件、任务用时、每道缝通过事件。 **诊断细分**（可选）：穿缝成功率（每道缝）、终点到达率。

### 7.2 碰撞率 collision rate

**主口径（业界 common practice，采纳）**：episode 级 CR `CR = 发生至少一次碰撞的 episode 数 / 总 episode 数`

- 与 §7.1 SR 的"全程无碰撞"成功条件天然对齐（碰撞即非成功）；但 CR 比 1−SR 更聚焦安全维度——超时/越界等非碰撞失败不计入 CR，可单独看"撞没撞"。
- 依据：AvoidBench、HUGE-Bench 等无人机导航 benchmark 均以此为主口径（"following common practice in collision-aware simulation benchmarks"）。

**辅助口径（可选，D3/D4 诊断用）**：CPK = 总碰撞次数 / 总飞行距离（次/km）

- 当要诊断"一个 episode 撞了几次"时启用，需引擎记录每次碰撞 + 总飞行距离（RVN-Bench 用法）。

**需引擎收集**：每 episode 是否碰撞（二值）、碰撞次数、碰撞时刻、飞行距离。

### 7.3 SPL（Success weighted by Path Length，Anderson 2018）

- 单次：`SPL_i = S_i × l_i / max(p_i, l_i)`
- 总体：`SPL = (1/N) Σ SPL_i`
- `S_i` = 成功（0/1，同 §7.1）；`l_i` = GT 最短可行穿缝路径（hidden GT，预计算）；`p_i` = 实际轨迹累积长度

**含义**：成功且走最短路径 = 1；成功但绕路 < 1；失败 = 0。 **需引擎收集**：实际轨迹累积长度、GT 最短路径。 **移动墙 caveat**：`l_i` 在移动墙下需重新定义（静态规划 vs 动态规划最短），待对齐。 **迷宫 caveat**：多缝错开（迷宫）时 `l_i` 需在所有合法穿缝顺序中求最短，本身即路径规划问题，待对齐（见 §9）。

### 7.4 配套指标

性能信号（控制频率 / 推理时延）是否计入评分，待对齐；先作运行监控。

> 本期所有题均为**执行型**（SUT 实际穿梭，对应 HB 等级 3–4 / 模版 L2+ 行为），按模版 M5 测执行型指标 SR/碰撞/SPL；不涉及模版 L1"只建议不执行"口径。
